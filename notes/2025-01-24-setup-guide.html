File Management Hub - Setup Guide

Created: January 24, 2025

Quick Setup Instructions

Follow these steps to get your File Management Hub up and running:

  1. Create GitHub Repository: Set up a new public repository on GitHub
  2. Upload Files: Copy all the File Management Hub files to your repository
  3. Enable GitHub Pages: Go to repository Settings → Pages → Select "main" branch
  4. Access Your Site: Visit https://yourusername.github.io/yourrepository

Folder Structure

Your repository should have this structure:

/
├── index.html (main page)
├── articles/ (news and articles)
├── notes/ (documentation and notes)
├── css/styles.css
├── js/
│   ├── main.js
│   ├── github-api.js
│   └── file-manager.js
└── README.md

Adding Content

To add files to your system:

  • Drop HTML, PDF, or Word files into the /articles/ or /notes/ folders
  • Use descriptive filenames with dates for better organization
  • Commit and push changes to GitHub
  • Files will automatically appear in your File Management Hub
Pro Tip: Include dates in your filenames like 2025-01-24-filename.pdf for automatic chronological sorting!

Customization

You can customize your File Management Hub by:

  • Editing css/styles.css to change the appearance
  • Modifying the repository detection in js/github-api.js
  • Adding new file types in js/file-manager.js

Troubleshooting

If files aren't showing up:

  1. Make sure your repository is public
  2. Check that files are in the correct folders
  3. Verify GitHub Pages is enabled
  4. Wait a few minutes for changes to propagate
